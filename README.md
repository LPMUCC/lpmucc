# LPMUCC — Let's Play Monopoly Using UCC™

Production codebase for lpmucc.com — the cryptic ARG game platform for the LAWLI Publishing three-book private law series by K. Lawli.

## Tech Stack
- **Next.js 14** App Router
- **Supabase** (database, auth, storage)
- **Stripe** (payments)
- **Resend** (Webmaster emails from webmaster@lpmucc.com)
- **Vercel** (deployment)

## Setup

### 1. Install dependencies
```bash
npm install
```

### 2. Environment variables
```bash
cp .env.example .env.local
```
Fill in all values. See `.env.example` for instructions on where to get each key.

### 3. Supabase setup
- Create a new project at supabase.com
- Go to SQL Editor
- Paste the entire contents of `supabase/migrations/001_initial_schema.sql`
- Run it — this creates all tables, policies, and triggers
- Go to Storage and create a private bucket called `vault-artifacts`

### 4. Run locally
```bash
npm run dev
```

### 5. Deploy to Vercel
- Push this repo to GitHub (already done)
- Go to vercel.com → New Project → Import from GitHub
- Select the `lpmucc` repository
- Add all environment variables from `.env.example`
- Deploy
- Add custom domain `lpmucc.com` in Vercel dashboard
- Update GoDaddy DNS: change nameservers to Vercel's nameservers

## Architecture

### Key routes
- `/` — Cryptic terminal homepage (hidden ENTER input appears after 8 seconds)
- `/activate` — Book code activation
- `/dashboard` — Player command center with 33-key grid
- `/submit` — Key submission with proof artifact upload
- `/vault` — Complete submission flow (33 keys + 180 day requirement)
- `/registry` — Private Banker Registry (public)
- `/webmaster` — Admin panel (requires admin role)
- `/phase/iv` — Easter egg 404 (second O in NOT uses alternate character)
- `/are/you/awake` — Satellite node (7 second delay)
- `/vault/33` — Black screen, single gold cursor (33 keys required)

### Anti-AI protections (all server-side)
1. Physical book codes required — generated by admin, printed at manufacturing
2. Single-use codes expiring every 90 days
3. Per-book prerequisite key gating (9 keys for Book 2, 21 for Book 3)
4. Immutable submission timestamps (database trigger prevents updates)
5. 180-day minimum account age for vault submission
6. Human review only — zero automated approvals
7. USPS tracking number verification
8. Annual cipher rotation (reveals Jan 1 to 20+ key holders only)
9. Annual proof format rotation
10. Pre-recorded audio challenge question

### Vault keys (Book One through Book Three)
See `src/lib/constants.ts` — VAULT_KEYS maps key numbers 1-33 to the correct words.
These are validated server-side in `/api/submit/route.ts` and never exposed to the client.

### Email system
All emails send from `webmaster@lpmucc.com` via Resend.
Templates are in `/src/lib/email-templates.ts`.
The Webmaster character never breaks the fourth wall.
